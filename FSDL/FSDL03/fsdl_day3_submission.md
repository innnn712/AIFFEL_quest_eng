### Q1. LLM 기반 시스템을 Accuracy만으로 평가할 수 없는 이유와 추가로 필요한 평가 차원 5가지

LLM 기반 시스템은 정답이 하나로 고정된 레이블이 없는 생성 모델이기 때문에 Accuracy로 계산이 불가능하다.  
따라서 아래와 같은 다차원적 평가가 필요하다:

1. **유창성 (Fluency)**: 자연스러운 한국어로 답변하는가?
2. **사실성 (Faithfulness)**: 없는 정보를 만들어내는 환각이 없는가?
3. **관련성 (Relevance)**: 질문에 맞는 답변인가?
4. **안전성 (Safety)**: 유해하거나 편향된 내용을 포함하지 않는가?
5. **의도 이해 (Intent Understanding)**: 사용자가 원하는 것을 정확히 파악했는가?

### Q2. LLM-as-a-Judge의 장점, 한계(편향), 완화 방법

**장점**  
인간 평가보다 훨씬 빠르고 저렴하여 빠른 반복이 가능하다. 인간 평가와의 상관관계가 0.7~0.9로 높다.

**한계 (편향)**  
길이 편향(Length Bias)이 존재하여, 답변 내용의 질과 무관하게 긴 답변에 더 높은 점수를 주는 경향이 있다.

**완화 방법**  
평가 프롬프트에 "길이에 관계없이 핵심만 평가하라"는 지시를 명시적으로 포함한다.

### Q3. 드리프트 3가지 유형과 챗봇 프로젝트에서 가장 가능성 높은 드리프트

**입력 드리프트 (Input Drift)**  
모델에 들어오는 데이터의 분포가 학습 시점과 달라지는 것.

**출력 드리프트 (Prediction Drift)**  
모델의 답변 분포가 학습 시점과 달라지는 것.

**성능 드리프트 (Performance Drift)**  
실제 성능 메트릭이 저하되는 것으로, 3가지 드리프트 중 가장 늦게 감지된다.

**챗봇 프로젝트 분석**  
가장 가능성 높은 드리프트는 **입력 드리프트**이다.  
이 챗봇은 학습 데이터가 고정적이고 사전학습 없이 처음부터 직접 학습한 모델이기 때문에,  
시간이 지나 사용자 질문 유형이 변하면 학습 데이터에 없는 표현/주제에 대해 즉시 엉뚱한 답변을 생성할 가능성이 높다.

### Q4. "모델은 시간이 지나면 썩는다"의 의미와 방지 시스템

ML 모델은 학습 시점의 현실 세계를 기반으로 만들어지기 때문에, 현실이 변하면 코드를 바꾸지 않아도 점점 쓸모없는 모델이 되어간다는 의미다.

**코로나19 사례**  
팬데믹 이전 데이터로 학습한 쇼핑몰 추천 모델은, 팬데믹 이후 소비 패턴이 마스크·운동복·홈오피스 용품으로 바뀌었음에도 여전히 여행용품·정장을 추천하는 쓸모없는 모델이 되었다.

**방지 시스템**  
- 드리프트 감지 시스템 (KS 검정 등으로 입력/출력 분포 변화 자동 감지)
- 자동 알림 (드리프트 감지 시 즉시 알림)
- 원인 파악 ("세상이 변한 건가? 파이프라인이 깨진 건가?" 구분)
- 대응: 경미한 경우 모니터링 강화, 심각한 경우 새 데이터 수집 후 재학습

## 2. ML 실행 계획서

### Part A. 기획 (Day 1 적용)

| 항목 | 내용 |
|------|------|
| **프로젝트 한 줄 정의** | 한국어 챗봇 대화 데이터를 학습한 Transformer seq2seq 모델이, 사용자의 질문 입력에 대해 답변을 생성하는 시스템 |
| **ML 도입 이유** | 사용자의 자연어 입력은 표현 방식이 무한히 다양하기 때문에, 규칙 기반(if-else)으로는 모든 경우를 커버하는 것이 불가능하다 |
| **접근법** | 처음부터 학습 (Scratch Training) |
| **비즈니스 메트릭** | 사용자와 자연스러운 대화가 3턴 이상 이어진다 |
| **ML 메트릭** | BLEU ≥ 10, BERTScore ≥ 0.75, Perplexity ≤ 50 |
| **베이스라인** | 검색 기반(Retrieval-based) — 학습 데이터에서 가장 유사한 질문의 답변을 반환 |

**리스크**
1. 학습 데이터에 없는 표현 입력 시 OOV(미등록어) 문제
2. 작은 모델 크기로 인한 성능 한계 (반복 답변, 엉뚱한 답변 생성 가능성)
3. Scratch Training이라 사전학습 모델 대비 다양한 표현 학습에 한계 → 향후 klue/bert 등 사전학습 모델로 교체 검토 필요

### Part C. 테스트와 품질 (Day 3 적용)

- 적용 가능: Yes (수동으로 1개 배치 반복 학습으로 구현)
- 10분 이내로 만들기 위해 조정할 것:
  - D_MODEL: 512 → 128 (모델 축소)
  - N_LAYERS: 2 → 1 (레이어 축소)
  - DROPOUT: 0.3 → 0.0 (규제 제거 — 암기 테스트 시 오버피팅 방해 요소 제거)

**행동 테스트**
- 불변성 테스트: 이모티콘이나 문장 부호가 추가되어도 같은 류의 답변이 나와야 한다
  - 예: "안녕" = "안녕~" = "안녕!!" → 모두 인사 관련 답변
- 방향성 테스트: 감정이 반대로 바뀌면 답변의 톤도 바뀌어야 한다
  - 예: "나 오늘 행복해" → 긍정적 답변 / "나 오늘 슬퍼" → 위로하는 답변
